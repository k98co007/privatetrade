## 작업 설명
LLD 2.0(특정 종목 선택 기능)을 기반으로 상세한 테스트 케이스를 작성. 모든 API 엔드포인트, UI 플로우, 에러 케이스를 포함한 테스트 시나리오.

## 테스트 범위
1. **API 테스트** (4개 엔드포인트)
   - POST /api/stocks/mode
   - POST /api/stocks/specific/add
   - GET /api/stocks/specific
   - DELETE /api/stocks/specific/{code}

2. **UI 테스트**
   - 모드 선택 (라디오 버튼)
   - 종목 추가/제거 (폼 입력)
   - 실시간 목록 업데이트

3. **통합 플로우**
   - 모드 전환 → 종목 선택 → 백테스팅 시작
   - 에러 케이스 (중복, 형식 오류, 최대값 초과)

4. **성능 테스트**
   - 100개 종목 추가/조회 응답 시간
   - 메모리 사용량

## 수락 기준
- [ ] 최소 50개 테스트 케이스 작성
- [ ] 각 TC별 전제조건, 시나리오, 기대 결과 명시
- [ ] API 응답 코드(200, 400, 500) 포함
- [ ] 커버리지 ≥ 80%
- [ ] 테스트 환경 담당자 검토 완료 (TICKET-012와 연동)

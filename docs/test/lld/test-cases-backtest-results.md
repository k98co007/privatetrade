| **소요 시간(예상)** | 5분 |
| **자동화 가능 여부** | 가능 |

#### 목적
백테스팅 시작 후 진행 상황이 실시간으로 UI에 표시되는지 확인하고, 진행률 업데이트가 정상적으로 작동하는지 검증

#### 사전 조건
- 웹 애플리케이션 접속 가능 상태
- 특정 종목 선택 페이지(specific-stock-selection.html) 로드 완료
- 특정 종목 1개 이상 선택됨
- 백테스팅 엔진 서버 정상 작동

#### 테스트 단계

1. 특정 종목 1개 이상 선택 확인
2. "백테스팅 시작" 버튼 클릭
3. **진행 섹션(progress-section) 표시 확인**
   - DOM에 `id="progress-section"` 요소 존재 확인
   - 요소의 display 속성이 "block" 또는 "flex"인지 확인
   - 요소가 화면에 가시적으로 표시되는지 확인
4. **초기 진행률 확인**
   - 진행률 표시 요소(`id="progress-percentage"`)가 "0%" 또는 "초기화" 상태로 시작하는지 확인
   - 진행 바(`id="progress-bar"`)의 width 속성이 0% 또는 초기값인지 확인
5. **진행률 증가 확인 (3회 이상)**
   - 1초 단위로 최소 3번 진행률 폴링
   - 각 폴링 시점의 진행률 값 기록: T1[%] → T2[%] → T3[%]
   - 진행률이 증가 또는 유지되는지 확인 (감소 불가)
   - 진행 바 애니메이션 확인 (CSS transition 적용 여부)
6. **진행 상태 메시지 표시 확인**
   - 진행 메시지 요소(`id="progress-message"`)에서 현재 작업 단계 메시지 확인
   - 메시지 예: "데이터 로드 중...", "신호 생성 중...", "거래 시뮬레이션 중..." 등
7. 최대 30분까지 진행 상황 모니터링 (또는 완료 시까지)

#### 예상 결과

| 상태 메시지 | ✓ 진행 중 메시지가 단계별로 업데이트됨 |
| 메시지 명확성 | ✓ 메시지가 사용자가 이해하기 쉬운 한국어로 표시 |

#### 사전 조건
- TC-BACKTEST-001 성공 완료
- 진행률이 100%에 도달
- 백테스팅 결과를 서버에서 수신
- 결과 계산 완료

#### 테스트 단계

1. **진행률 100% 확인**
   - 진행률 요소에서 "100%" 표시 확인
   - 진행 바가 전체 길이로 채워진 상태 확인
2. **결과 섹션 표시 확인**
   - DOM에 `id="results-section"` 요소 존재 확인
   - 요소의 display 속성이 "block" 또는 "flex"인지 확인
   - 진행 섹션이 적절히 숨겨지는지 확인
3. **완료 배지 표시 확인**
   - 완료 배지(`class="completion-badge"`) 표시 확인
   - 배지 색상이 초록색(성공 상태)인지 확인
   - 배지에 "✓ 완료" 또는 "완료" 텍스트 확인
4. **5개 성과 지표 렌더링 확인**

   a) **총 수익률 (Total Return)**
   - 요소 ID: `id="metric-total-return"`
   - 예상값: 45.32%
   - 표시 형식: "45.32%"
   - 색상 코딩: 양수이므로 초록색(#00AA00 또는 유사)
   
   b) **샤프 지수 (Sharpe Ratio)**
   - 요소 ID: `id="metric-sharpe-ratio"`
   - 예상값: 1.85
   - 표시 형식: "1.85"
   - 색상 코딩: 양수이므로 초록색
   
   c) **최대 손실률 (Max Drawdown)**
   - 요소 ID: `id="metric-max-drawdown"`
   - 예상값: -12.5%
   - 표시 형식: "-12.5%"
   - 색상 코딩: 음수이므로 빨강색(#DD0000 또는 유사)
   
   d) **거래 횟수 (Trade Count)**
   - 요소 ID: `id="metric-trade-count"`
   - 예상값: 247
   - 표시 형식: "247회" 또는 "247"
   - 색상 코딩: 중립(검정 또는 기본색)
   
   e) **승률 (Win Rate)**
   - 요소 ID: `id="metric-win-rate"`
   - 예상값: 56.8%
   - 표시 형식: "56.8%"
   - 색상 코딩: 양수이므로 초록색

5. **메타데이터 표시 확인**
   - 백테스트 ID 표시: "ID: [고유 ID]" 형식
   - 완료 시간 표시: "완료: 2026-02-08 14:32:15" 형식
   - 또는 소요 시간 표시: "소요 시간: 5분 23초"

6. **숫자 포맷 검증**
   - 소수점 자릿수 확인 (%.%% 형식 확인)
   - 천 단위 구분 확인 (거래 횟수가 1000 이상일 경우)
   - 통화 기호 확인 (해당되는 경우)

7. **레이아웃 검증**
   - 5개 지표가 모두 가시적으로 표시된 위치 확인
   - 지표들이 겹치거나 잘려있지 않은지 확인
   - 메타데이터가 명확한 위치에 표시되는지 확인

### TC-BACKTEST-003: 에러 처리

| 속성 | 값 |
|------|-----|
| **TC 번호** | TC-BACKTEST-003 |
| **제목** | 에러 처리 |
| **우선순위** | High |
| **복잡도** | 4 |
| **소요 시간(예상)** | 10분 |
| **자동화 가능 여부** | 부분 가능 |

#### 목적
API 오류, 타임아웃, 네트워크 장애 등 예상되는 에러 상황에서 애플리케이션이 안정적으로 처리하는지 검증

#### 사전 조건
- 웹 애플리케이션 접속 가능 상태
- 특정 종목 선택 완료
- 백테스팅 엔진 서버 시뮬레이션 가능 상태
- 브라우저 개발자 도구 네트워크 탭 접근 가능

#### 테스트 단계

1. **정상 시나리오 실행 (베이스라인)**
   - 정상적으로 백테스팅 1회 완료
   - 결과가 정상 표시되는지 확인

2. **API 호출 실패 시뮬레이션 (테스트 케이스 A)**
   - 백테스팅 시작
   - 네트워크 패킷 조작 또는 모의 서버를 통해 API 응답 실패 시뮬레이션
   - HTTP 상태 코드: 500 (Internal Server Error) 또는 503 (Service Unavailable)
   - 에러 응답 확인:
     - **에러 메시지 표시**: 빨강색 배경의 에러 메시지 팝업 또는 섹션 표시
     - **메시지 내용**: "백테스팅 실패: 서버 연결 오류" 또는 유사한 명확한 메시지
     - **메시지 유지 시간**: 최소 3초 이상 표시
     - **닫기 버튼**: 에러 메시지 닫기 버튼 또는 X 버튼 제공

3. **타임아웃 자동 중단 확인 (테스트 케이스 B)**
   - 백테스팅 시작
   - 서버 응답을 의도적으로 지연 또는 차단
   - 30분 경과 대기 (또는 타임아웃 설정값 확인)
   - **타임아웃 처리**:
     - 자동으로 요청 중단되는지 확인
     - 에러 메시지 표시: "요청 타임아웃: 30분 초과" 또는 유사 메시지
     - 진행 섹션이 정지되는지 확인
     - UI가 응답 불가능 상태가 되지 않는지 확인

4. **결과 조회 실패 및 재시도 확인 (테스트 케이스 C)**
   - 백테스팅 진행 중 결과 조회 API 실패 시뮬레이션
   - **재시도 로직**:
     - 1차 시도 실패 후 자동 재시도 확인
     - 재시도 간격: 1초 또는 설정값 확인
     - 최대 재시도 횟수: 3회 이하
     - 재시도 횟수가 UI에 표시되는지 확인 (선택사항): "재시도 1/3", "재시도 2/3", "재시도 3/3"
   - **최대 재시도 초과 후**:
     - 에러 메시지 표시: "최대 재시도 횟수 초과" 또는 유사 메시지
     - "다시 시도" 또는 "초기화" 버튼 제공

5. **네트워크 종료 시나리오 (테스트 케이스 D)**
   - 백테스팅 진행 중 네트워크 종료 (오프라인으로 전환)
   - **동작 확인**:
     - 애플리케이션이 crashed 또는 frozen되지 않는지 확인
     - 에러 메시지가 표시되는지 확인
     - 네트워크 복구 후 재시도 가능한지 확인

6. **에러 복구 테스트**
   - 에러 발생 후 "다시 시도" 또는 "초기화" 버튼 클릭
   - 새로운 백테스팅 시작 가능한지 확인

- 특정 종목 선택 완료

1. **중복 실행 방지 확인 (테스트 케이스 A)**

   - "백테스팅 시작" 버튼 클릭 직후 버튼 상태 확인:
     - 버튼이 **disabled** 상태로 변환되는지 확인
     - 버튼 텍스트가 변경되는지 확인 (예: "진행 중..." 또는 "대기 중...")
     - 버튼 색상이 회색 또는 비활성 색상으로 변경되는지 확인
   - 버튼이 disabled 상태인 동안 다시 클릭 시:
     - 백테스팅이 중복 실행되지 않는지 확인
     - 콘솔에 에러 메시지가 표시되지 않는지 확인

2. **새로고침 후 이전 결과 조회 확인 (테스트 케이스 B)**
   - 백테스팅 완료 및 결과 표시된 상태에서 페이지 새로고침 (F5 또는 Ctrl+R)
   - **새로고침 후**:
     - 결과 섹션이 유지되는지 확인 (완전히 사라지지 않음)
     - 5개 성과 지표가 여전히 표시되는지 확인
     - 진행 섹션이 사라지는지 확인
   - **데이터 저장 확인**:
     - 로컬 스토리지 또는 세션 스토리지에서 결과 데이터 확인
     - 백엔드 데이터베이스에서 결과 조회 가능 여부 확인

3. **"초기화" 버튼으로 상태 리셋 확인 (테스트 케이스 C)**
   - 백테스팅 완료 및 결과 표시된 상태
   - "초기화" 또는 "리셋" 버튼 클릭 (해당 버튼이 없으면 "새로운 백테스팅" 버튼)
   - **초기화 후 상태**:
     - 결과 섹션이 숨겨지는지 확인
     - 진행 섹션도 숨겨지는지 확인
     - "백테스팅 시작" 버튼이 다시 enabled 상태로 변환되는지 확인
     - 페이지가 백테스팅 시작 전 상태로 완전히 돌아가는지 확인
     - 로컬 스토리지 데이터가 삭제되는지 확인 (필요한 경우)

4. **새로운 백테스팅 시작 가능 확인 (테스트 케이스 D)**
   - 초기화 후 특정 종목을 다시 선택
   - "백테스팅 시작" 버튼이 fully enabled 상태인지 확인
   - 버튼 클릭으로 새로운 백테스팅 시작 가능한지 확인
   - 이전 결과와 새로운 결과가 혼동되지 않는지 확인:
     - 새로운 백테스팅 ID가 생성되는지 확인
     - 새로운 완료 시간이 기록되는지 확인
     - 새로운 성과 지표가 표시되는지 확인

5. **메모리 누수 확인 (선택사항)**
   - 브라우저 개발자 도구의 메모리 탭에서 메모리 사용량 모니터링

테스트 실행 전에 다음 항목을 확인해야 합니다:

- [ ] 웹 서버 정상 작동 (localhost 또는 배포 서버)
- [ ] 백테스팅 엔진 서버 정상 작동
- [ ] 데이터베이스 연결 정상 작동
- [ ] 테스트 종목 데이터 준비
- [ ] 브라우저 개발자 도구 확인 가능 상태
- [ ] 네트워크 시뮬레이션 도구 준비 (TC-BACKTEST-003용)

### 테스트 환경

| 항목 | 사양 |
|-----|------|
| **브라우저** | Chrome 120+, Firefox 121+, Edge 120+ |
| **OS** | Windows 10/11, macOS 12+, Ubuntu 22+ |
| **화면 해상도** | 1920x1080 (메인), 375x667 (모바일 테스트용) |
| **네트워크** | 안정적인 인터넷 (최소 5Mbps) |
| **테스트 데이터** | test-data/mock-stocks.json 사용 |

```markdown

- **결과**: ✓ PASS / ✗ FAIL
- **소요 시간**: 5분 23초

```

## 부록

### A. 자동화 테스트 작성 가이드

향후 이 테스트 케이스들을 자동화할 경우 다음 도구 사용을 권장합니다:

- **Selenium** 또는 **Cypress**: E2E 자동화
- **Jest** 또는 **Mocha**: 단위 및 통합 테스트
- **Lighthouse**: UI 성능 및 접근성 검증

### B. 색상 코드 참고

| 색상명 | 16진 코드 | RGB | 사용처 |
|--------|----------|-----|--------|
| 초록색 | #00AA00 | (0, 170, 0) | 양수 지표 |
| 빨강색 | #DD0000 | (221, 0, 0) | 음수 지표, 에러 |
| 파랑색 | #0066FF | (0, 102, 255) | 진행 바 |
| 회색 | #EEEEEE | (238, 238, 238) | 진행 바 배경 |

### C. 성과 지표 정의

| 지표명 | 계산식 | 해석 |
|--------|---------|------|
| 총 수익률 (Total Return) | (최종 자산 - 초기 자산) / 초기 자산 × 100% | 양수: 수익, 음수: 손실 |
| 샤프 지수 (Sharpe Ratio) | (평균 수익 - 무위험률) / 수익 표준편차 | 높을수록 위험 대비 수익 우수 |
| 최대 손실률 (Max Drawdown) | (최고점 - 최저점) / 최고점 × 100% | 음수: 최악의 손실 |
| 거래 횟수 (Trade Count) | 총 거래 건수 | 많을수록 활발, 적을수록 보수적 |
| 승률 (Win Rate) | 수익 거래 / 전체 거래 × 100% | 높을수록 우수 |
